2. CODE STANDARDS & TERMINAL PROTOCOLS
--------------------------------------
2.1  Node Scripts Must Be ESM
     Since this project operates with "type": "module" in package.json, all 
     temporary or auxiliary Node.js scripts must use ES Module syntax 
     (e.g., use "import fs from 'fs';" instead of "const fs = require('fs');").

2.2  Surgical Patch Scripts
     When modifying large source files, prefer writing temporary Node.js 
     scripts (utilizing `fs.readFileSync` and `.replace()`) over full file 
     overwrites to prevent command truncation or buffer loss.
     CRITICAL: Always append the cleanup command (e.g., "&& rm ./patch.js") 
     on the exact same execution line to keep the repository untracked space clean.
     Actively clean up and delete any other transient scripts or patch helpers 
     immediately after execution to prevent workspace clutter.

2.3  Zsh Compatibility & Escaping
     The development environment is on macOS running Zsh. Never use raw, unescaped 
     exclamation marks (!) inside shell string variables, as Zsh parses them 
     as history expansion operators and will fail execution. 

2.4  Zero Bash Comments
     Never include hash symbols (#) inside any active terminal or shell code 
     blocks. Copy-pasting shell commands containing hash lines directly into 
     macOS interactive Zsh terminals triggers immediate parsing errors or partial 
     execution. Keep shell blocks strictly functional and describe operations 
     in the accompanying markdown text instead.

2.5  Strict Structured File Patching
     When modifying CSS, JSON, or complex TypeScript layouts with nested brackets, 
     do not use lazy regex matching (e.g. "[\s\S]*?") to replace large blocks, 
     as it easily misaligns closing braces and corrupts files. Perform full 
     overwrites for small-to-medium files or strict literal string matching 
     for surgical updates.

2.6  Unicode and Escapes in Bash
     When writing inline scripts via terminal heredoc, never use raw unicode 
     escapes (such as "\u00A0") if they risk being parsed by the terminal shell 
     as literal invisible characters. Instead, use runtime code conversions 
     like `String.fromCharCode(160)` to safely inject special characters.

2.7  Quoted Starting Delimiters (Shell Protection)
     Always wrap the starting heredoc delimiter in single quotes (e.g., 
     `cat << 'EOF' > file` or `cat << 'ENDOFSCRIPT'`). Unquoted delimiters 
     allow the shell to perform immediate variable expansion ($variable) and 
     backtick execution before writing, stripping or corrupting raw JS, TS, 
     and CSS structures.

2.8  Delimiter Syntax and Formatting
     The terminating delimiter (e.g., EOF or ENDOFSCRIPT) must sit alone on its 
     own line with zero leading or trailing whitespace. Any leading spaces or 
     tabs will prevent the terminal from recognizing the block boundary, 
     causing terminal lockup or truncation.

2.9  Strict TypeScript Constraints
     Always comply with strict compiler rules (such as 'noUnusedLocals', 'noUnusedParameters',
     and '@typescript-eslint/no-explicit-any') by stripping unused variables, constants, or imports,
     and completely avoiding explicit 'any' type declarations. If a general or dynamically resolved
     type is required, store references as 'unknown' and perform type assertions, narrow type guards,
     or implement clean generic abstractions to ensure type-safe interfaces. When modifying or expanding
     strongly typed configurations, nested structures, or read-only literals (e.g., 'as const'), verify
     that all symmetrically accessed keys exist across all related record types or union branches to
     prevent compile-time property errors (TS2339).

    function GlobalScrollbar({ lenisInstance }) {

          const trackHeight = trackRef.current.clientHeight;
          const scrollDelta = (deltaY / trackHeight) * maxScrollY;

            lenisInstance.scrollTo(targetY, { immediate: true });
          } else {
            window.scrollTo(0, targetY);
          }
        };

          setIsDragging(false);
          document.body.style.userSelect = '';
          document.removeEventListener('pointermove', handlePointerMove);
          document.removeEventListener('pointerup', handlePointerUp);
        };

        document.addEventListener('pointerup', handlePointerUp);
      };
